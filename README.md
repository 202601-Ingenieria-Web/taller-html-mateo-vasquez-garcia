# Explorando la PokeAPI

Este es un pequeño proyecto cuyo objetivo fue **explorar una API abierta**, en este caso la **PokeAPI**, y demostrar la integración de la misma en un proyecto de frontend utilizando **HTML5**.

La verdad fue un ejercicio bastante entretenido. Me sirvió para explorar y entender varias cosas sobre el consumo de APIs. Además, recientemente he estado volviendo a jugar algunos juegos de Pokémon, así que ya estaba un poco enganchado con el tema.

Mientras exploraba más a fondo la API y sus posibilidades, descubrí cosas que al inicio no tenía contempladas, como las formas alternativas de algunos Pokémon (megas, Gmax, entre otras). Implementarlas terminó siendo un pequeño reto porque me generaron algunos errores inesperados durante el desarrollo, pero al final fue parte del aprendizaje y de lo interesante del ejercicio.

En general fue una actividad bastante entretenida y útil para practicar la integración de APIs en el frontend.

# Ejecución del proyecto

Antes de ejecutar el proyecto, primero debes **descargar el repositorio**, ya sea clonándolo o descargándolo directamente.

Después de eso, hay varias formas sencillas de ejecutarlo:

### 1. Usando Visual Studio Code + Live Server

Esta es la forma que yo utilicé.

1. Abre el proyecto en Visual Studio Code.
2. Instala la extensión 'Live Server' si no la tienes.
3. Haz clic derecho sobre el archivo `index.html`.
4. Selecciona "Open with Live Server".

Esto abrirá automáticamente el proyecto en tu navegador.

### 2. Abrir directamente en el navegador

Esta es probablemente la forma más universal.

1. Busca el archivo `index.html` dentro del proyecto.
2. Haz 'clic derecho' sobre él.
3. Selecciona 'Abrir con'.
4. Escoge el navegador de tu preferencia.

### 3. Arrastrar el archivo al navegador

Otra opción muy simple:

1. Abre tu navegador favorito.
2. Busca el archivo `index.html` en el explorador de archivos.
3. Arrástralo al navegador.

El proyecto se abrirá automáticamente.

Las opciones 2 y 3 no requieren tener un IDE instalado, por lo que son las más rápidas si solo quieres ver el proyecto funcionando.
